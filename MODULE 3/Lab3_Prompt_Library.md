# Lab 3: Building a Prompt Library for BCA Use Cases

**Target Audience:** BCA Students  
**Module:** Prompt Engineering  
**Duration:** 1 Hour

---

## 🎯 Objective
By the end of this lab, you will:
- Understand the importance of maintaining a personal **Prompt Library**
- Create effective, reusable prompts for common BCA tasks
- Build your own Prompt Library (Markdown file) that you can use throughout your studies and career

---

## 📝 Why Build a Prompt Library?

- Good prompts take time to craft
- Reusing well-tested prompts saves time and gives consistent results
- Helps you become more efficient with AI tools
- Professional skill used by developers, analysts, and content creators

---

## 🚀 Task: Build Your Personal Prompt Library

### Instructions
Create a Markdown file named `My_Prompt_Library.md` and add at least **8 high-quality prompts** useful for BCA students.

---

### Example Prompts (You can copy and improve them)

#### 1. Summarizing Notes / Research Papers
```text
You are an expert BCA professor. Summarize the following text into clear bullet points. Use simple language. Focus on key concepts, definitions, and examples.

Text: [Paste your notes here]
```

#### 2. Explaining Complex Concepts (Feynman Technique)
```text
Act as an expert Computer Science tutor. Explain the concept of [Insert Concept, e.g., Pointers in C / Object-Oriented Programming] to me as if I am a complete beginner. Use a real-world analogy and break down the explanation step-by-step.
```

#### 3. Code Generation with Best Practices
```text
Act as a Senior Software Developer. Write a [Insert Language, e.g., Python / Java] program to [Insert Task, e.g., sort an array using QuickSort]. 

Requirements:
1. Include comments explaining the core logic.
2. Follow best naming conventions.
3. Handle basic edge cases (like an empty array).
```

#### 4. Code Debugging and Error Fixing
```text
Act as an expert Code Reviewer. My [Insert Language] code below is throwing this error: "[Paste Error Message]". 

Please:
1. Identify the exact line causing the bug.
2. Explain WHY it is happening in simple terms.
3. Provide the corrected code.

Code:
[Paste your buggy code here]
```

#### 5. Database & SQL Query Generation
```text
Act as a Database Administrator. I need an SQL query for a PostgreSQL database.

Schema:
[Paste your table structures/schema here, e.g., Table 'Users' with columns id, name, email]

Task: Write a query to [Insert Task, e.g., find the top 5 users who registered this month]. Please explain the logic behind any JOINs or subqueries used.
```

#### 6. Interview Preparation (Mock Interview)
```text
Act as a strict Technical Interviewer for a [Insert Role, e.g., Junior React Developer] position. Ask me one technical interview question at a time. Wait for my answer before giving feedback and asking the next question. Do not provide the answer unless I ask for it. Let's start with the first question.
```

#### 7. Project Ideation and Planning
```text
Act as an expert Tech Lead. I am a BCA student looking to build a final year project. 
My skills are: [Insert Skills, e.g., HTML, CSS, JavaScript, Node.js].

Suggest 3 unique mini-project ideas. For each idea, provide:
1. A catchy project name.
2. A brief 2-sentence description.
3. The exact tech stack I should use.
4. Three main features to implement.
```

#### 8. Automatic Documentation Generation
```text
Act as a Technical Writer. Please generate a professional `README.md` file for my project based on the code provided below. 

The README should include:
- A brief project description
- Prerequisites / Requirements
- Step-by-step installation instructions
- Usage examples

Code:
[Paste your main script or project structure here]
```

---

## 💡 Best Practices for Your Prompt Library
- **Use Placeholders:** Use brackets like `[Insert Code Here]` so you know exactly what needs to be swapped out when you reuse the prompt.
- **Categorize:** Group your prompts into sections like "Coding", "Studying", and "Career Prep".
- **Refine:** If a prompt stops working well, tweak it and update your library. A prompt library is a living document!
